Solana blockchain

A practical mental model: what it is, why it’s fast, and how you actually use it.

Solana for beginners • 10 slides
Slide 02 Foundation

First: what is a blockchain?

A blockchain is a shared database that many computers keep in sync — so no single party controls the “truth.”

  • Ledger: a history of transactions (who sent what to whom).
  • Consensus: the network agrees on the next valid update.
  • Public verification: anyone can verify the state from the data.
Slide 03 What you’re learning

So… what is Solana?

Solana is a blockchain designed for high throughput and low fees, making it good for apps that need fast interactions.

  • Fast: blocks confirm quickly.
  • Cheap: small fees for most transactions.
  • Programmable: apps run via on-chain programs (smart contracts).
Beginner shortcut

Think of Solana as a global computer

It stores accounts (data + balances) and runs programs that update those accounts.

Accounts = data Programs = logic Transactions = updates
Slide 04 Network roles

Who makes Solana work?

You’ll interact through a wallet, but under the hood multiple roles keep the chain running.

  • Wallet: your keys + approvals. It signs transactions.
  • RPC node: a server you query to read/send transactions.
  • Validators: machines that vote on blocks and keep the network honest.
Slide 05 State model

Accounts, SOL, and tokens

In Solana, most things are an account — even token balances are stored in accounts.

  • SOL: the native token used for fees and payments.
  • Token accounts: hold balances of SPL tokens (USDC, etc).
  • Data accounts: store app data (profiles, escrow, game state).
Why this matters

Programs don’t “own” your wallet

A program can only change accounts it’s allowed to touch — and you approve that via a transaction.

Slide 06 How change happens

What is a transaction?

A transaction is a set of instructions that say “update these accounts using this program.”

  • 1) Build: the app creates a transaction with instructions.
  • 2) Sign: your wallet approves (signs) with your private key.
  • 3) Send + confirm: it’s broadcast and becomes final after confirmation.
Slide 07 Smart contracts

What is a Solana program?

A program is on-chain code that reads/writes accounts. It’s like a backend API that the whole network runs.

Mental model

Programs are stateless

The program itself doesn’t store your app data. The accounts store the data. Programs only validate and update them.

if user_signed && account.owner == program_id { verify(rules); account.data.counter += 1; emit(event); }
Slide 08 Costs + performance

Why are Solana fees usually low?

Fees depend on network demand, but Solana is built for throughput and efficient execution.

  • Fee: small SOL paid to process your transaction.
  • Rent (simplified): accounts must keep a minimum balance to stay stored.
  • Speed: quick confirmations make apps feel “web2 fast.”
Slide 09 Don’t get rekt

Beginner safety checklist

Most losses happen from approvals and links — not from “Solana being hacked.”

  • Verify addresses: don’t paste random “token mints” or program IDs.
  • Read prompts: a wallet approval can be a transfer, not a login.
  • Start on devnet/testnet: learn with play money first.
